1. Purpose & Scope

This register satisfies TOR Section 5.3 (Risk Register & Remediation Backlog). It replaces the bullet-list summary in the initial submission (Annex C) with a structured per-finding register including: unique risk ID, priority, affected module(s), finding description, root cause, evidence source, likelihood, impact, risk score, recommended corrective action, and evidence required to close.

⚠ Scope note: All findings in this register are based on static source code analysis only. Likelihood and impact ratings reflect the worst-case scenario derivable from code evidence. Ratings marked with an asterisk (*) should be validated against runtime evidence when pre-production access is available. See Annex I (Constraints Register) for a full list of testing not yet executable.
