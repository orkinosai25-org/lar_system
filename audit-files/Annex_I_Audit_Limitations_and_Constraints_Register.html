1. Purpose & TOR Reference

This register satisfies TOR Section 5.6 (Audit Limitations & Constraints Register) — a mandatory deliverable. It records every condition, restriction, or access gap that bounded the scope, depth, or confidence of findings in this audit engagement. It is produced to enable LAR to make informed decisions about residual risk and to identify what additional testing is required before any finding can be closed with full evidence.

Findings cited in the main audit report (v7.3.3) and Annex H (Static Analysis Evidence Log) must be read in conjunction with this register. Any finding whose evidence source is restricted by a constraint listed below should be treated as a tentative finding pending confirmation through the testing specified in the "Required to Close" column.

2. Environment & Access Constraints

The following access and environment limitations applied throughout this engagement.

C-001 — Critical No Access to Pre-Production Environment
Constraint: No login credentials, VPN access, or SSH access to any pre-production, staging, or production environment were provided to the auditor at any point during the engagement.
Impact on findings: Zero runtime evidence (PHP error logs, Apache/Nginx logs, application logs, session logs) could be collected. All findings are from static source code analysis only. No booking scenario, repricing test, API trace, or TTL test could be executed.
TOR reference: TOR §5.5 requires an evidence pack including logs, traces, booking references, and reproducible test steps — these cannot be produced without environment access.
Required to close: Provision of a fully-configured pre-production environment matching the production topology, with sandbox Amadeus GDS credentials, for a minimum of 5 business days of controlled testing access.
C-002 — Critical No GDS / Amadeus Sandbox Credentials
Constraint: No Amadeus API sandbox or test environment credentials were provided. No credentials for any external GDS or supplier API were provided.
Impact on findings: The three contracted verticals — Flights, Hotels, and Cars (TOR §3.1) — could not be tested end-to-end. Fare parity, PNR integrity, HK/HL/UC confirmation status, TTL validation, repricing, and stale-cache behaviour could not be verified.
Required to close: Amadeus test environment credentials + all applicable supplier API sandbox access.
C-003 — Critical No Runtime Application Logs Available
Constraint: Server is not running in the analysis environment. No /var/log/apache2/error.log, /var/log/php-fpm.log, CodeIgniter application logs, or payment gateway transaction logs are accessible.
Impact on findings: Cannot verify actual error output in production, cannot identify runtime-only failures (e.g., silent booking failures, silent payment gateway errors), cannot confirm frequency or recurrence of parse errors under live load.
Required to close: Export of last 90 days of application and web server error logs from production and pre-production environments, with PII redacted.
C-004 — Significant No Database Schema or Sample Data Access
Constraint: No database access was provided. No schema export, ERD, or anonymised sample dataset was made available.
Impact on findings: SQL injection risk assessment is limited to code-level pattern identification only. Cannot confirm exploitability with actual data. Cannot assess data residency, retention, or PII exposure. PCI DSS and GDPR compliance assessment is incomplete without schema knowledge.
Required to close: Provision of anonymised database schema export and a representative anonymised sample dataset in a controlled read-only sandbox.
C-005 — Significant No Network / Infrastructure Access
Constraint: No access to web server configuration (Apache/Nginx), firewall rules, TLS/SSL configuration, CDN configuration, or deployment infrastructure was provided.
Impact on findings: HTTPS enforcement, HSTS, security headers (Content-Security-Policy, X-Frame-Options, etc.), TLS version, and certificate status cannot be assessed. WAF presence and configuration cannot be verified.
Required to close: Server configuration file exports (sanitised) and access to run testssl.sh or equivalent against the pre-production HTTPS endpoint.
C-006 — Significant No CI/CD Pipeline Access or Artefacts
Constraint: No GitHub Actions workflow configuration, no Playwright test reports, no existing CI artefacts were made available for review.
Impact on findings: Cannot confirm whether any automated regression tests exist. Cannot verify that existing tests would catch the identified regressions. Cannot assess release gating quality.
Required to close: Export of all CI workflow definitions and the last 10 CI run reports (pass/fail with artefact links).

4. Data & Evidence Constraints

D-001 — Significant Revenue / Financial Impact Figures Cannot Be Validated
Constraint: No booking volumes, payment transaction counts, conversion rate data, or revenue data were provided or accessible.
Impact: Any specific dollar revenue-at-risk figure included in the initial submission (e.g. $6.8M) has been removed from the revised deliverables (Annex F Revised) as it cannot be derived without booking volume assumptions, data sources, and a documented methodology per TOR §3.3.
Required to close: Provision of anonymised monthly booking counts, payment success/failure rates, and average booking values — or explicit written instruction from LAR to use an industry-standard estimate methodology with stated assumptions.
D-002 — Significant No PNR or Booking Reference Records
Constraint: No PNR records, booking references, customer journey logs, or session traces were provided.
Impact: PNR integrity, atomicity, and lifecycle audit trail findings (TOR §5.x Flights) cannot be substantiated with specific booking evidence. The finding that PNR integrity is at risk is based solely on the presence of stub code and parse errors in the flight model — not on observed booking failures.
Required to close: Access to anonymised booking records in the pre-production environment, or controlled execution of end-to-end booking scenarios with captured traces.
D-003 — Moderate No Previous Audit Reports or Baseline
Constraint: No previous audit reports, security assessments, or technical debt registers were provided as a baseline.
Impact: Cannot distinguish newly introduced vulnerabilities from historical technical debt. Cannot assess whether previously identified issues have been remediated. All findings are treated as first-observation.
Required to close: Provision of any prior technical assessment, pen-test reports, or security scan outputs.
