# LAR System — Audit Report Folder

This folder contains audit and error reports automatically generated by the CI/CD workflows.

## Contents

| File Pattern | Source Workflow | Description |
|---|---|---|
| `vertical-error-report-*.txt` | `vertical-tests.yml` | Aggregated PHP syntax, critical file, DB config, and CodeIgniter structure checks across all 6 verticals |
| `LAR_Post_Deploy_Audit_Report_*.txt` | `post-deploy-audit-report.yml` | Comprehensive post-deployment audit report combining vertical checks, Azure runtime logs, and integration test results |

## How Reports Are Generated

### Vertical Error Reports (`vertical-tests.yml`)
Runs on every push to `main`, every PR targeting `main`, and weekly (Sundays 02:00 UTC).
Checks each vertical (B2C, Agent, Supplier, Supervision, Services, UltraLux) for:
- PHP syntax errors
- Critical file/directory existence
- Database config security (env vars, no hardcoded credentials)
- CodeIgniter structure and TODO/FIXME counts

The aggregated report is committed here when running on the `main` branch.

### Post-Deploy Audit Reports (`post-deploy-audit-report.yml`)
Triggers automatically after each successful "Vertical Integration Tests" run on `main`,
weekly (Tuesdays 07:00 UTC), and can be triggered manually.
Combines outputs from:
- Vertical PHP checks
- Azure deployment logs
- Functional integration test results

The full audit report is committed here and also retained as a GitHub Actions artifact for 90 days.

## Reviewing Reports

1. Open the latest `LAR_Post_Deploy_Audit_Report_*.txt` for the most recent comprehensive audit.
2. Open the latest `vertical-error-report-*.txt` for the most recent per-vertical error summary.
3. For older runs, download artifacts directly from [GitHub Actions](../../actions).
